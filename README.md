# ˹ᴜʟᴛʀᴀ ᴍᴜꜱɪᴄ˼ — بوت موسيقى تيليغرام

بوت تيليغرام لتشغيل الموسيقى والفيديو في المكالمات الصوتية للمجموعات (Voice Chat)، مبني على
**Pyrogram (kurigram)** و **PyTgCalls** و **MongoDB**، مع دعم تعدد المساعدين (Assistants)،
قائمة انتظار، وضع القناة، إحصائيات الأكثر استماعاً، ودعم متعدد اللغات.

---

## 1. متطلبات التشغيل

### البرمجيات الأساسية

| المتطلب | الوصف |
|---|---|
| Python | الإصدار 3.10 فأعلى (يُفضّل 3.13 كما في `Dockerfile`) |
| FFmpeg | لازم لمعالجة وتشغيل الصوت/الفيديو |
| MongoDB | قاعدة بيانات لتخزين الإعدادات، اللغة، القائمة السوداء، الإحصائيات... |
| حساب تيليغرام مساعد (Userbot) | لازم لتشغيل الصوت في المكالمة الصوتية (Voice Chat) |
| بوت تيليغرام | يُنشأ عبر [@BotFather](https://t.me/BotFather) |

### بيانات يجب الحصول عليها قبل التشغيل

1. **API_ID و API_HASH**: من [my.telegram.org](https://my.telegram.org/apps)
2. **BOT_TOKEN**: من [@BotFather](https://t.me/BotFather)
3. **OWNER_ID**: آيدي حسابك على تيليغرام، يمكن الحصول عليه من [@userinfobot](https://t.me/userinfobot)
4. **LOGGER_ID**: آيدي مجموعة خاصة (سالبة القيمة) يضاف البوت إليها لتسجيل الأحداث
5. **MONGO_DB_URI**: رابط الاتصال بقاعدة بيانات MongoDB (يمكن استخدام [MongoDB Atlas](https://www.mongodb.com/atlas) مجاناً)
6. **STRING_SESSION**: جلسة الحساب المساعد (Userbot)، تُولَّد عبر بوت توليد الجلسات مثل @StringFatherBot

### تثبيت المكتبات

```bash
pip install -r requirements.txt
```

أهم المكتبات المستخدمة (بإصدارات ثابتة لضمان التوافق):

```
kurigram~=2.2.22                                      # واجهة Pyrogram لبروتوكول تيليغرام
py-tgcalls~=2.2.11                                     # تشغيل البث في المكالمات الصوتية
pymongo~=4.17.0                                        # الاتصال بقاعدة بيانات MongoDB
pillow~=12.2.0                                         # معالجة الصور (الصور المصغّرة)
aiohttp~=3.13.5                                        # طلبات HTTP غير متزامنة
psutil~=7.2.2                                          # مراقبة استهلاك الموارد (CPU/RAM)
pytgcrypto~=1.2.12                                     # تسريع طبقة التشفير لتيليغرام
py-yt-search~=0.6.2                                    # البحث في يوتيوب بدون مفتاح API
python-dotenv~=1.2.2                                   # تحميل متغيرات البيئة من ملف .env
yt-dlp[default] @ git+https://github.com/yt-dlp/yt-dlp.git   # تحميل الصوت/الفيديو من يوتيوب
```

> **ملاحظة:** `yt-dlp` يُثبَّت مباشرة من مستودعه الرسمي على GitHub لضمان الحصول على آخر إصلاحات
> كشف الحماية من يوتيوب، بدل الاعتماد على نسخة PyPI التي قد تتأخر في التحديث.

## 2. طريقة إعداد ملف `.env`

1. انسخ القالب الجاهز:

```bash
cp sample.env .env
```

2. افتح ملف `.env` وعبّئ القيم الإلزامية التالية (البوت **لن يعمل** بدونها، حسب فحص `config.py`):

```env
API_ID=123456
API_HASH=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
BOT_TOKEN=123456:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
LOGGER_ID=-1001234567890
MONGO_DB_URI=mongodb+srv://user:pass@cluster.mongodb.net/
OWNER_ID=123456789
STRING_SESSION=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

3. (اختياري) متغيرات إضافية يمكن تركها بدون تعبئة، وسيستخدم البوت قيمتها الافتراضية:

| المتغير | الوصف | الافتراضي |
|---|---|---|
| `COOKIE_URL` | رابط ملف كوكيز يوتيوب (للفيديوهات المحظورة عمرياً/جغرافياً) | غير مفعّل |
| `STRING_SESSION2` / `STRING_SESSION3` | جلسات مساعدين إضافيين لتوزيع الحمل على مجموعات أكثر | غير مفعّل |
| `DURATION_LIMIT` | الحد الأقصى لمدة الأغنية بالدقائق | `300` |
| `QUEUE_LIMIT` | الحد الأقصى لعدد الأغاني في قائمة الانتظار | `30` |
| `PLAYLIST_LIMIT` | الحد الأقصى لعدد الأغاني المستوردة من قائمة تشغيل يوتيوب | `20` |
| `AUTO_END` | إيقاف البث تلقائياً عند انتهاء قائمة الانتظار (`True`/`False`) | `False` |
| `AUTO_LEAVE` | خروج المساعد تلقائياً من المكالمة بعد عدم نشاط (`True`/`False`) | `False` |
| `THUMB_GEN` | توليد صورة مصغّرة مخصصة لكل أغنية (`True`/`False`) | `True` |
| `VIDEO_PLAY` | تفعيل أوامر تشغيل الفيديو `/vplay` (`True`/`False`) | `False` |
| `VIDEO_MAX_HEIGHT` | أقصى دقة فيديو يتم تحميلها (مثل `720` أو `1080`) | `1080` |
| `EXCLUDED_USERNAMES` | أسماء مستخدمين مستثناة من إشعارات منشن الأدمن (مفصولة بمسافة) | فاضي |
| `SUPPORT_CHAT` / `SUPPORT_CHANNEL` | روابط الدعم التي تظهر في رسائل الخطأ/الترحيب | روابط افتراضية |

### تشغيل البوت

```bash
python3 -m UltraMusic
```

أو باستخدام Docker:

```bash
docker build -t ultramusic .
docker run --env-file .env ultramusic
```

## 4. ملاحظات مهمة حول هذا التحديث

- **نظام اللغة (`core/lang.py`)**: تم إصلاحه ليقرأ تفضيل اللغة المخزَّن لكل مجموعة في
  MongoDB (عبر `db.get_lang` / `db.set_lang` الموجودين مسبقاً في `core/mongo.py` ويُستخدمان
  من `/language`)، مع رجوع تلقائي (**fallback**) لأي مفتاح ترجمة غير موجود في اللغة المختارة
  إلى نسخته الإنجليزية، بدل توقف البوت بخطأ `KeyError`.
- **إحصائيات الأكثر استماعاً (`/topchat` `/topglobal` `/topuser`)**: كانت لا تعمل فعلياً قبل
  هذا التحديث لأن `db.increment_track()` لم يكن يُستدعى من أي مكان. تم ربطها الآن داخل
  `play.py` مباشرة بعد كل عملية تشغيل ناجحة.
- **أوامر الكتم (`/mute` و `/unmute`)**: جديدة بالكامل، وتعمل عبر دالتين أُضيفتا في
  `core/calls.py` (`tune.mute` / `tune.unmute`) بنفس أسلوب `pause`/`resume` الموجود مسبقاً.
- **⚠️ تنبيه بخصوص ازدواج أوامر القائمة السوداء:** يحتوي المشروع حالياً على نظامين منفصلين
  لحظر المجموعات: `plugins/admin-controls/blacklist.py` (الجديد، بأوامر `/blacklist`
  `/whitelist` `/blacklisted`) و `plugins/settings/blacklist.py` (موجود سابقاً، بأوامر
  `/blacklistchat` `/whitelistchat` `/blacklistedchat`). كلاهما يستخدمان نفس قاعدة البيانات
  (`db.add_blacklist` / `db.get_blacklisted`) فلا يوجد تعارض تقني أو تصادم في أسماء الأوامر،
  لكن يُفضَّل لاحقاً حذف أحد الملفين لتجنّب الازدواج الوظيفي.

## 5. الدعم

في حال مواجهة أي مشكلة، راجع رسائل الخطأ التي يعرضها البوت (تتضمن غالباً السبب والحل المقترح)،
أو راجع رابط الدعم المُعرَّف في متغير `SUPPORT_CHAT` ضمن ملف `.env`.
